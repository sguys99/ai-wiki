## 소개

검색 증강 생성(RAG)은 대규모언어모델이 외부 지식을 활용해 정확한 답변을 생성하는 기법입니다. 그러나 기존 RAG 시스템은 "텍스트 중심으로 설계되어 있어, 실제 문서에서 빈번하게 등장하는 다이어그램, 표, 수식, 차트 같은 비텍스트 요소들을 효과적으로 처리하지 못하는 한계가 있었습니다."

홍콩대학교 데이터 인텔리전스 랩(HKUDS)이 이 문제를 해결하기 위해 **RAG-Anything**을 공개했습니다.

---

## RAG-Anything 개요

RAG-Anything은 "LightRAG(EMNLP 2025)를 기반으로 구축된 올인원 멀티모달 RAG 시스템"입니다. 이 프레임워크는:

- 텍스트, 이미지, 표, 수식을 단일 인터페이스로 처리
- PDF, Office 문서, 이미지를 포함한 다양한 형식 지원
- `pip install raganything` 명령으로 간단하게 설치 가능

---

## 핵심 기술적 도전 과제

### 1. 통합 멀티모달 표현
"서로 다른 정보 유형을 매끄럽게 통합하면서도 각 모달리티의 고유한 특성과 모달리티 간 관계를 보존"해야 합니다.

### 2. 구조 인지 분해
"복잡한 레이아웃을 지능적으로 파싱하면서 공간적, 계층적 관계를 유지"하는 레이아웃 인지 파싱이 필요합니다.

### 3. 크로스-모달 검색
"서로 다른 모달리티 사이를 탐색하고 그 상호 연결을 추론할 수 있는 정교한 메커니즘"을 요구합니다.

---

## RAG-Anything의 핵심 기여

### 이중 그래프 구축
"크로스-모달 지식 그래프와 텍스트 기반 지식 그래프를 별도로 구축한 뒤 엔티티 정렬로 융합하는 전략"을 사용합니다.

### 크로스-모달 하이브리드 검색
"그래프의 구조적 탐색과 임베딩 공간의 의미 유사도 매칭을 결합하여 다중 홉 추론과 의미 검색을 동시에 수행"합니다.

### 모달리티 인지 질의 처리
질의에 포함된 "figure", "table", "equation" 같은 단서를 분석해 모달리티 선호를 추론합니다.

### 긴 문서 강건성
문서가 길어질수록 베이스라인 대비 성능 격차가 증가합니다.

---

## 멀티모달 파이프라인 구성

### 1. 멀티모달 파싱 엔진
PDF, DOCX, PPTX 등 다양한 형식을 입력받아 텍스트, 이미지, 수식, 표을 모두 처리합니다.

### 2. 크로스-모달 지식 그래프 구축
각 콘텐츠 단위에서 검색용 설명과 그래프 구축용 엔티티 요약을 생성하며, 문맥을 고려합니다.

### 3. 그래프 융합 및 인덱스 생성
멀티모달 그래프와 텍스트 기반 지식 그래프를 엔티티 이름으로 정렬하여 통합합니다.

### 4. 크로스-모달 하이브리드 검색
구조적 탐색과 의미 유사도 매칭을 병행하여 후보를 검색하고 재정렬합니다.

### 5. 멀티모달 응답 합성
최종 텍스트 컨텍스트와 시각 자료를 비전-언어 모델에 입력하여 응답을 생성합니다.

## 케이스 스터디

### 다중 패널 그림 해석
t-SNE 시각화처럼 여러 서브패널을 가진 그림에서, "패널, 축 라벨, 범례, 캡션을 노드로 하고 의미 관계를 엣지로 가지는 시각 레이아웃 그래프"를 구축합니다.

### 금융 표 탐색
"행 헤더, 열 헤더(연도), 데이터 셀, 단위를 노드로 하고 관계를 명시적으로 모델링하여, 유사한 용어가 반복되는 재무 표에서도 정확히 정보를 추출"합니다.

```bash
pip install raganything
```

### 기본 사용 예시

```python
import asyncio
from raganything import RAGAnything

async def main():
    # RAG-Anything 초기화
    rag = RAGAnything(
        llm_model_func=your_llm_func,
        vision_model_func=your_vision_func
    )

    # 멀티모달 문서 인덱싱
    await rag.process_document_complete(
        file_path="research_paper.pdf",
        output_dir="./rag_output"
    )

    # 멀티모달 질의 응답
    result = await rag.query(
        "논문의 주요 실험 결과와 관련 그림을 설명해줘",
        mode="hybrid"
    )
    print(result)

asyncio.run(main())
```

## 한계 및 향후 과제

시스템의 한계:
- **텍스트 중심 검색 편향**: 시각 정보가 명시적으로 요구되는 질의에서도 텍스트 소스를 선호
- **경직된 공간 처리**: 셀 병합, 비표준 칼럼 경계 등 표준 레이아웃을 벗어난 문서 처리에 제한
